# 추가 성능 향상 전략 (실험 검증 기반, v2)

> 대규모 실험 후 작성. **✅검증됨 / ❌무효 / ☐미시도(ROI 표기)**. 현재 best 대비 추가 향상 방안.
> **메타 결론**: 대부분 스테이지가 천장 근접. 가장 가치있는 미시도 = Stage2 **합성-사전학습→실데이터 파인튜닝**.

## 현재 best
| 스테이지 | best | 한계 요인 |
|---|---|---|
| Stage1 | PR-AUC **0.446** (Maha+교호작용피처) | 양성 7건 = 통계 천장 |
| Stage2 | macro-F1 **0.935** (SE-ResNet 6앙상블+보정) | Loc/Scratch 본질적 난이도 |
| Stage3-A | mAP **0.739**(bestV2) → YOLO11m 진행 | 합성데이터 |
| Stage3-B | Grad-CAM(실데이터) | GT 마스크 없음(정량화 제약) |

## 검증 결과 요약 (무엇이 통했나)
- ✅ **Stage2**: 깊이+SE attention(0.86→0.93), 앙상블(+0.006), 증강, per-class 임계보정
- ✅ **Stage1**: 선별 교호작용 피처(0.30→0.45), 공분산 모델(Maha/GMM), rank-앙상블
- ❌ **무효(정직)**: 입력 패딩/해상도, balanced sampling, **SimCLR·MAE 자기지도**, **DINOv2 전이**, TTA, CBAM(=SE), 전수 쌍조합(차원의저주)

## Stage 1 — 추가 향상 (천장 근접, ROI 낮음)
| 방법 | ROI | 근거/기대 |
|---|---|---|
| **Ledoit-Wolf shrinkage 공분산**(정규화 Maha) | 中 | 고차원 교호작용을 과적합 없이 → 전수조합 살릴 수도(현재 붕괴) |
| **교호작용 피처 선택**(MI/L1) | 中 | 선별>전수 입증됨 → top-k 자동선택으로 추가 이득 가능 |
| **물리기반 피처**(전력=VI, 식각효율 등 도메인) | 中 | 결함=조합 메커니즘 정합 |
| PCA-whiten + Maha | 低 | EllipticEnvelope와 유사 |
| 더 많은 라벨 결함 확보 | ★(불가) | 7건이 본질 한계 — 데이터가 답 |
→ **권장**: shrinkage 공분산 + 피처선택. 단 0.45가 사실상 천장(7건). 성능보다 **신뢰도(k-fold CI)** 가 정직.

## Stage 2 — 추가 향상 (가장 유망한 미시도 존재)
| 방법 | ROI | 근거/기대 |
|---|---|---|
| **★ 합성(MixedWM38) 사전학습 → 실데이터 파인튜닝** | **中~高** | zero-shot 전이는 실패했지만 **파인튜닝**은 다름. 합성으로 표현 초기화 후 실데이터 적응 → 난클래스 보강 가능. **미시도** |
| **per-class focal / LDAM / logit-adjustment** | 中 | Loc(0.86)·Scratch(0.87) 표적. balanced sampling은 실패했지만 손실레벨 보정은 더 정교 |
| 앙상블 다양성(ConvNeXt·다른 width 멤버 추가) | 中 | 6앙상블이 통했으니 이질적 멤버로 추가 다양성 |
| WM-811K **라벨노이즈 정제** | 中 | 알려진 노이즈 → 정제 시 상한↑ (수작업 비용) |
| Mixup/CutMix·hierarchical(none→type) | 低 | 한계적 |
| 데이터 더(none↑) | 低 | **결함은 이미 전수 사용** → 효용 작음(확인됨) |
→ **권장**: ① 합성→실 파인튜닝(가장 유망) → ② per-class 손실보정 → ③ 이질 앙상블. **목표 0.94+**.
→ 한계: Loc↔Edge-Loc/Center, Scratch↔Loc 혼동은 **본질적**(저밀도·산발) → ~0.94 천장 추정.

## Stage 3 — 추가 향상
**A 검출(ELLIMAC, 합성)**
| 방법 | ROI | |
|---|---|---|
| YOLO11m/l + 증강·imgsz↑ | 中 | 진행중(11m 200ep) |
| YOLO 앙상블·TTA | 低中 | 검출 표준 부스트 |
| 라벨 전수 정제 | 中 | 노이즈 제거 |
→ 단 **합성이라 실전 일반화 가치 제한** → 스킬 데모로 충분.

**B 위치(Grad-CAM, 실데이터)**
| 방법 | ROI | |
|---|---|---|
| **Grad-CAM++ / Score-CAM / Eigen-CAM** | 中 | 선명도↑ |
| 약지도 위치 정량화(GT 없음→대리지표) | 低 | WM-811K 위치 GT 부재가 제약 |
→ 강한 Stage2 모델(SE-ResNet) CAM + ++ 정도가 현실적.

## Stage 4 (신설) — 성능보다 "범위" 확장
| 후보 | 가치 | Stage2 되먹임 |
|---|---|---|
| LLM 통합 원인리포트(공정+패턴+위치) | 中(차별화) | 종합 |
| 희귀패턴 생성(diffusion) | 低(희귀 이미 NF 1.0) | 효용 작음 |
| 멀티태스크 통합(공유 백본) | 中 | 연구성 |
→ Stage4는 **성능보다 데모/서사 가치**. 우선순위 낮음.

---

## 종합 우선순위 (추가 향상 ROI)
1. **Stage2 합성→실 파인튜닝** (유일한 高ROI 미시도) ★
2. **Stage2 per-class 손실보정**(Loc/Scratch)
3. **Stage1 shrinkage 공분산 + 피처선택**
4. Stage2 이질 앙상블(ConvNeXt)
5. Stage3-A YOLO11m(진행) / Stage3-B Grad-CAM++
6. Stage4(서사 확장, 성능 아님)

> **정직한 천장 인식**: Stage1 0.45(7건)·Stage2 0.94(본질 난이도)가 한계 추정.
> 추가 향상은 **+1~2% 수준**이며, 핵심 가치는 이미 달성(0.30→0.45, 0.86→0.935)+정직한 negative 자산.
