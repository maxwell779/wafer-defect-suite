# 성능 향상 연구 전략 (Stage 1–4)

> 실측 실험에 근거. **✅통함 / ❌안통함 / ☐미시도** 로 표기. 각 방법에 기대효과·근거.
> 핵심 교훈: **단순 튜닝(입력·샘플링·시드앙상블)은 한계, 구조적 변화(깊이+attention)가 돌파.**

---

## Stage 1 — 공정 센서 이상탐지 (Meruva)
현재: Maha/OCSVM/LOF ~**0.295±0.03**(30-seed) > AE(DL) 0.215. **양성 7건 = 통계 천장.**

| 방법 | 상태 | 기대 | 근거 |
|---|---|---|---|
| AD 모델 확장 (ECOD·COPOD·KDE·GMM·kNN·DeepSVDD) | ☐ | 소폭 | PyOD 계열, 모델 다양성 |
| **AD 점수 rank-앙상블** (Maha+LOF+OCSVM) | ☐ | 안정성↑(분산↓) | 동률 모델 결합 시 robust |
| 피처 엔지니어링 (pressure/etch 비율·교호작용) | ☐ | 소폭 | 분리신호 3변수 강화 |
| stratified k-fold(결함 포함)+bootstrap CI | ☐ | 신뢰구간 정확화 ★ | 7건 단일split 노이즈 |
| PU-learning / 합성 양성(SMOTE) 지도보조 | ☐ | 불확실 | 7건이라 과적합 위험 |
- **현실**: 7 양성이 본질 한계. **권장 = rank-앙상블 + k-fold CI**(성능보다 신뢰도). 성능 상향 여지 작음(정직).

---

## Stage 2 — 웨이퍼 패턴 ★플래그십 (최대 투자)
현재: **SE-ResNet 0.928(보정)** (CNN 0.890 → +0.038). 난클래스 Scratch 0.85·Loc 0.83·Edge-Loc 0.90.

**이미 검증된 것**
- ✅ **깊이 + SE attention**(0.890→0.928) — 최대 레버
- ✅ 회전/플립 증강(+0.02) · per-class 임계보정(+0.01)
- ❌ 입력 패딩/해상도↑, ❌ balanced sampling, ❌ SimCLR 자기지도, ❌ DINOv2 전이(0.33)

**연구 메뉴 (ROI 순)**
| 방법 | 상태 | 기대 | 근거 |
|---|---|---|---|
| **SE-ResNet 앙상블**(3 init) | 🔄 진행 | +0.5~1% | 동일아키 다양성 |
| **CBAM(공간+채널 attention)** / CoordConv | ☐ | 위치패턴(Center vs Loc)↑ | 웨이퍼는 위치 의미 큼 |
| **ConvNeXt-tiny / ResNeXt** (더 강한 백본) | ☐ | +1~2% 가능 | SE-ResNet이 통했으니 연장 |
| **MAE 자기지도**(63.8만 미라벨, 마스킹복원) | ☐ | 난클래스↑ 가능 | SimCLR(대조) 실패 → MAE(복원)가 격자에 적합 |
| **로짓조정/LDAM/CB-Focal**(불균형 손실) | ☐ | NF·Donut↑ | balanced sampling보다 정교 |
| **TTA**(회전4× 추론평균) | ☐ | +0.3~0.5% | 무학습 부스트 |
| **극좌표 변환**(r,θ) 입력 | ☐ | Ring/Center 회전불변↑ | 웨이퍼 원형 구조 |
| 라벨 데이터 전량 사용(normal_cap↑·전체 결함) | ☐ | 특이도↑ | 현재 일부만 |
| **lot 5-fold CV** | ☐ | 신뢰구간 ★ | 단일 split 보완 |
| 멀티라벨 분해(MixedWM38) 위 기법 적용 | ☐ | 분해성능↑ | 합성 멀티라벨 |
- **권장**: ResNet앙상블 → CBAM/ConvNeXt → MAE → TTA·CV. **목표 0.93~0.94.**

## Stage 3 — 결함 위치 (검출 A + Grad-CAM B)
현재: A ELLIMAC bestV2 mAP@0.5 **0.739**(합성) / B Grad-CAM(정성).

| 방법 | 상태 | 기대 | 근거 |
|---|---|---|---|
| **YOLO11 정식 학습**(정제 라벨, 증강, 큰 모델) | ☐ | bestV2 비교/상회 | 현재 벤치마크만 |
| 라벨 전수 정제(폴리곤→bbox·cls6) + 검증 | ☐ | mAP↑ | 노이즈 제거 |
| **Grad-CAM++ / Score-CAM / Eigen-CAM** | ☐ | 위치 선명도↑ | CAM 개선 계열 |
| **SE-ResNet 기반 CAM**(더 강한 Stage2 모델) | ☐ | 위치 정확도↑ | 강모델=선명 CAM |
| 위치 **정량화**(pointing-game·IoU) | ☐ | 수치화 ★ | 정성→정량 |
| 약지도 위치학습(WSOL: CAM-loss) | ☐ | 불확실 | 박스 없이 위치 |
- **권장**: B를 SE-ResNet CAM + Grad-CAM++로 강화 & 정량화(실데이터 가치). A는 YOLO11 학습으로 보강(합성 한계 명시).

---

## Stage 4 (신설 제안) — 무엇을 더하면 좋은가
현재 3-스테이지. "stage4"로 가치 높은 후보:

| 후보 | 내용 | 다른 스테이지와 연결 |
|---|---|---|
| **A. 희귀패턴 생성(diffusion/GAN)** ★ | 웨이퍼맵 결함패턴 생성 → 희귀(NF/Scratch/Loc) 증강 | **Stage2 난클래스 직접 개선**(피드백 루프) |
| B. 통합 의사결정/리포트(LLM) | 공정값+패턴+위치 → 원인추론·조치 리포트 | Stage1·2·3 종합, 버스바 리포트 경험 |
| C. 수율 예측·가상계측 | 공정→수율 회귀 | Stage1 확장 |
| D. MLOps/모니터링·드리프트 | 배포·재학습·드리프트 탐지 | 운영 |
- **추천 Stage4 = A(희귀패턴 생성)**: 가장 연구적이고 **Stage2 성능을 되먹임**으로 끌어올림(생성→증강→난클래스↑). 정직성: 합성생성은 전이검증 필수(우리가 이미 그 함정을 안다 = 강점).

---

## 종합 우선순위 (ROI)
1. **Stage2 ResNet 앙상블 + TTA** (진행중, 즉시 +)
2. **Stage2 CBAM/ConvNeXt + MAE** (구조적 추가 상향, 0.93+)
3. **Stage3-B SE-ResNet CAM + Grad-CAM++ 정량화** (실데이터 위치 가치)
4. **Stage1 rank-앙상블 + k-fold CI** (신뢰도)
5. **Stage4-A 희귀패턴 생성 → Stage2 되먹임** (연구·차별화)
6. Stage3-A YOLO11 학습 (합성, 부록)

> 일관 원칙: **구조적 변화 > 단순 튜닝**(실험으로 입증), 모든 향상은 **leak-free·다중seed·정직보고**.
