## 2. 단계별 성능 향상 여정

### Stage 1 — 0.31 → 0.81
`base 0.31` → `선별 교호작용 0.45` → `표적 메커니즘(저압×고온/식각) 0.78` → `지도/비지도 하이브리드 0.81`
- 결함 7건이 전부 **압력↓ + 고온/식각** 조합 → "결함=센서 비정상 조합"을 표적 피처로 직접 인코딩한 것이 결정적.
- **천장 확정**: 1000조합 랜덤서치(reps=60 CI±0.034)·스태킹·ECOD/COPOD·PU·SMOTE·MI 전부 ~0.81을 못 넘음 → 양성 7건이 통계적 한계(표준편차 ±0.31이 압도).

### Stage 2 — 0.36 → 0.935
`합성 학습 0.985` → `합성→실제 전이 0.36(붕괴)` → `실데이터 직접 0.86` → `CNN강화+보정 0.90` → `SE-ResNet 0.91` → `6-앙상블 0.935`
- 구조적 변화(깊이 + SE attention) + 앙상블이 결정적(0.86→0.935).
- **천장 규명**: cleanlab 라벨정제(오라벨 1.3% 제거) Δ-0.005 → 한계는 고칠 수 있는 노이즈가 아니라 **Loc↔Edge-Loc 본질 모호성**. 손실(tversky/ldam/smoothbce)·Mixup/CutMix·GeM/maxavg 풀링·ViT·dilated·width96 전부 0.935 못 넘음.
- **600조합 메가서치(프록시 4-shard 병렬→상위 12 풀학습)**: 단일모델 best **test 0.9274** (val 0.9232). 단일모델 천장도 ~0.927로 확인 → 0.935는 6-앙상블+보정으로만 도달. (상세 `docs/overnight/RESULTS_stage2mega.md`)
- **앙상블 고도화(negative)**: 20개 다양 멤버로 mean/greedy/가중/스태킹(LogReg) 전부 비교(leak-free val→test, ±TTA). 최고 = 가중 **0.9304**(no-TTA), 스태킹 0.925. **어느 방법도 기존 6-앙상블+TTA 0.935를 못 넘음** → 다양성·메타러너로도 천장(Loc↔Edge-Loc 라벨 모호성) 불변 재확인. (`src/stage2_wafermap/ensemble2.py`)

### Stage 3 — 0.739 → 0.753
`bestV2 0.739` → `YOLO11m 0.753`(채택) / `YOLO11l 0.755`(동률 → 효율적 11m 유지)
- 폴리곤 라벨 bbox 정제 + 잘못된 cls6 18줄 제거 후 11n/m/l 비교.
- ELLIMAC은 **실제 칩표면 사진**(Roboflow 증강)으로 웨이퍼맵과 도메인이 달라 검출 스킬 데모로 활용.

## 3. 핵심 발견 / 교훈

1. **합성만으론 실전에서 무너진다** — 합성 학습 0.985가 실데이터에서 0.36으로 붕괴(정상 오탐 0.957). 도메인 갭을 정량·인과로 규명하고 실데이터 학습으로 회복.
2. **도메인 지식 > 무차별 ML** — Stage1에서 결함 메커니즘을 표적 피처로 인코딩(0.78)한 것이 전수 조합(차원의 저주, 0.07~0.22)보다 압도적.
3. **"복잡 ≠ 좋음" (정직한 negative)** — 1000조합 튜닝·스태킹·PU·ECOD/COPOD(Stage1), cleanlab·손실·Mixup·ViT·width(Stage2), YOLO11l(Stage3) 등 수많은 고급 기법이 천장을 못 넘음을 **CI로 정량 규명**.
4. **엄격한 평가** — lot 단위 leak-free 분할, 임계값 val-only, 양성 소수는 reps 반복 CI, per-class·불균형 지표(macro-F1/mAP/PR-AUC).

## 4. 천장의 본질 (왜 더 못 올리나)

| Stage | 천장 | 원인 | 근거 |
|---|---|---|---|
| 1 | ~0.81 | 양성 7건 통계 한계 | 1000조합·reps60도 동일, ±0.31 분산 |
| 2 | ~0.935 | Loc↔Edge-Loc 라벨 모호성 | cleanlab 정제 Δ-0.005(노이즈 아님) |
| 3 | ~0.75 | 합성/다른 도메인 | 11l 더 키워도 0.755 동률 |

→ 추가 향상은 **데이터 본질**(더 많은 결함 라벨·명확한 경계)에 달려 있으며, 모델·튜닝으로는 한계. 이를 회피하지 않고 정량적으로 보고한 것이 이 프로젝트의 가치.
